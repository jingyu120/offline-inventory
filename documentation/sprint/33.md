### Sprint 33: Comprehensive Transaction Auditing & Identity Readiness (Phase 6)

Execute the following architectural overhauls to establish a tamper-evident, highly resilient audit trail that anticipates a future Role-Based Access Control (RBAC) login flow. Adhere strictly to Drizzle ORM standards and the offline-first paradigms defined in `ai-instructions.md`.

**1. Immutable Event Sourcing Schema (Shared/Backend):**

- Target `@burma-inventory/shared-types`.
- Create a new Drizzle schema model named `AuditEvents`.
- Fields must include: `event_id`, `trace_id`, `actor_id` (UUID string), `device_id` (UUID string), `entity_type` ('ORDER', 'SHOP', 'INVENTORY'), `action` ('CREATE', 'UPDATE', 'DELETE', 'OVERRIDE'), `previous_state` (JSONB), `new_state` (JSONB), `gps_coordinates`, and `created_at`.
- Implement an append-only architecture. Updates to `Orders` or `Shops` must simultaneously commit a row to `AuditEvents` within a single Drizzle database transaction block.

**2. Abstracted Actor Context (Mobile & Backend):**

- **Mobile:** Create an `ActorService` utilizing `react-native-mmkv`. For current development, this service must initialize and expose a deterministic, mocked `actor_id` (e.g., `dev-user-uuid-0001`) and a persistent `device_id`.
- **Backend:** Create a globally scoped `ActorInterceptor` in NestJS. For now, it extracts the `x-actor-id` and `x-device-id` from incoming request headers and injects them into the execution context.
- Every audit log and sync payload must dynamically call these services. This guarantees zero-friction swapping when real JWT authentication is implemented later.

**3. Cryptographic Tamper-Evident Logging (Mobile):**

- Natively hash the payload of every `AuditEvent` on the mobile device utilizing a fast SHA-256 utility.
- The cryptographic signature must combine the current event data, the `actor_id`, and the hash of the _previous_ event in the local SQLite table to prevent silent offline tampering.
- The backend `sync.controller.ts` must recalculate this hash chain upon receipt. If broken, flag the payload as `COMPROMISED`.

**4. Distributed Trace Context Propagation (Mobile & Backend):**

- When a checkout or audit workflow is initiated on the mobile device, generate a unique `x-trace-id` (UUIDv4).
- This `trace_id`, alongside the `actor_id`, must be attached to the SQLite draft cart, the resulting image uploads (Viber screenshots), the AI worker queue jobs, and the final Drizzle `AuditEvents` commit.

**5. Network-Aware Queue Prioritization (Mobile):**

- Update the background sync engine (`syncEngine.ts`).
- If the device detects a highly degraded 2G/EDGE network, immediately pause the `ImageUploadQueue`.
- Dynamically reallocate all bandwidth to flush the lightweight `AuditEvents` and text payloads first to ensure financial telemetry survives intermittent connections.
